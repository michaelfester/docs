When we click on a list item, we get taken to that item’s details screen where we can customize it using components.

{% figure src="/docs/reference/ux/details-layout/details-screen.gif" caption="Details screen" /%}

But we an also use the **Details** layout for the top level of our tabs and structure that with components.

{% figure src="/docs/reference/ux/details-layout/details-layout.gif" caption="Details layout" /%}

One of the most common reasons we would do this is if we don't want our tab to be a list of items, but rather a page with different things on it, like a home screen.

{% figure src="/docs/reference/ux/details-layout/dl-applied-to-first-tab.png" caption="Details layout applied to the first tab to create a 'Welcome page'" /%}

We can also use the **Details layout** to create a single screen into which we place multiple components, **Inline Lists** and **Relations** to create a truly unique layout for our screen.

{% figure src="/docs/reference/ux/details-layout/details-layout2.gif" caption="Details layout" /%}
